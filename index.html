feature section

User Authentication

All users are authenticated before accessing the resources that are available on the application. Authentication is done using JWT. The token is created and verified in a middleware function and then a response is generated using cookie which is made available by the browser. The cookie is attached to every request and parsed to the server using an npm package called cookie-parser. All users are required to signup before using the resources.

User Authorization

There are 3 categories of users available in the application. These users are identified by their roles in the userSchema - 'admin', 'home-owner' and 'student'. There is only one user assigned with the admin privileges and have total control of all activities in the application including CRUD operations. The home-owners can perform CRUD actions but only on resources that is created by the user while the student can only access the resources and make a choice of the apartment to rent. The actions of each user is clearly defined in the project.

Project Inspiration

The inspiration for this project was born out of the frustration people pass through to get apartment especially those that relocate from one city to another. This project is particularly targeted at University Students and NYSC participants. These are the category of people that frequently have to relocate to other cities and one of the major challenges that they usually face is the problem of getting decent accommodation at a fair price. Most times, many people fall victims to fraudsters who pose as agents. This project will drastically reduce those anomalies.

This is a portfolio project for ALX Africa - School of Software Engineering.
